# RSVP-форма
Язык: русский

## Purpose
Форма подтверждения присутствия с выбором гостей, напитка, трансфера, аллергии и пожеланий.

## Requirements

### Requirement: Переключатель присутствия
Форма MUST содержать переключатель из двух кнопок для указания, будет ли гость присутствовать.

#### Scenario: Выбор «Да, буду»
- ДАНО отображается RSVP-форма
- КОГДА гость нажимает «Да, буду»
- ТОГДА становятся видимыми выбор количества гостей, переключатель трансфера и выбор напитка
- И кнопка отправки становится активной

#### Scenario: Выбор «Нет, не смогу»
- ДАНО отображается RSVP-форма
- КОГДА гость нажимает «Нет, не смогу»
- ТОГДА при отправке передаются guests=0, drink='Не пью', transfer='no'
- И дополнительные поля (гости, трансфер, напиток) скрыты

### Requirement: Выбор количества гостей
При подтверждении присутствия гость MUST иметь возможность выбрать количество гостей от 1 до 4.

#### Scenario: Выбор 2 гостей
- ДАНО гость нажал «Да, буду»
- КОГДА гость выбирает «2 гостя» в выпадающем списке
- ТОГДА значение guests=2 отправляется в API

### Requirement: Переключатель трансфера
Гость MUST указать, нужен ли ему трансфер («Да» / «Нет»).

#### Scenario: Выбор трансфера
- ДАНО гость нажал «Да, буду»
- КОГДА гость нажимает «Да» на переключателе трансфера
- ТОГДА значение transfer=yes отправляется в API

### Requirement: Выбор напитка
Гость MUST выбрать предпочитаемый напиток из выпадающего списка, заполняемого из API.

#### Scenario: Загрузка напитков из API
- ДАНО RSVP-форма монтируется
- КОГДА компонент создаётся
- ТОГДА вызывается `fetchDrinks()`
- И опции выпадающего списка заполняются полученными названиями напитков
- И первый напиток предвыбран по умолчанию

### Requirement: Поле аллергии
Гость MUST указать наличие аллергии через переключатель (Да/Нет). Если выбрано «Да», появляется текстовое поле для ввода продуктов.

#### Scenario: Аллергия включена
- ДАНО гость нажал «Да, буду»
- КОГДА гость нажимает «Да» на переключателе аллергии
- ТОГДА появляется текстовое поле для ввода продуктов

### Requirement: Поле пожеланий
Форма MUST содержать текстовое поле для пожеланий или комментариев.

#### Scenario: Заполнение пожеланий
- ДАНО форма отображается
- КОГДА гость вводит текст в поле «Пожелания»
- ТОГДА текст отправляется в API как параметр wishes

### Requirement: Защита от ботов
Форма MUST реализовывать honeypot + временна́я метка для защиты от ботов.

#### Scenario: Honeypot заполнен
- ДАНО бот заполняет скрытое honeypot-поле
- КОГДА форма отправляется
- ТОГДА отправка молча отклоняется
- И в консоль выводится «🤖 Бот: заполнено honeypot-поле»

#### Scenario: Форма отправлена слишком быстро
- ДАНО пользователь открывает страницу
- КОГДА форма отправляется менее чем через 2500мс после загрузки
- ТОГДА отправка отклоняется
- И в консоль выводится «🤖 Бот: форма отправлена слишком быстро»

### Requirement: Серверная валидация
Форма MUST отправлять подтверждение через POST. Бэкенд MUST проверять все поля перед записью.

#### Scenario: Валидация на сервере
- ДАНО форма отправлена с `guests=5`
- КОГДА бэкенд проверяет значение
- ТОГДА возвращается ошибка «Некорректное количество гостей»
- И данные не записываются

### Requirement: Отправка
Форма MUST отправить данные подтверждения в API (через POST) и отобразить состояние успеха.

#### Scenario: Успешная отправка
- ДАНО форма заполнена валидными данными
- КОГДА гость нажимает «Отправить»
- ТОГДА вызывается `submitConfirmation(guests, drink, transfer, wishes, allergy)`
- И в случае успеха форма заменяется карточкой «Вы подтвердили приглашение!»
- И `weddingConfirmed` устанавливается в `true` в localStorage

#### Scenario: Ошибка отправки
- ДАНО форма отправлена
- КОГДА API возвращает ошибку или недоступен
- ТОГДА под кнопкой отправки отображается сообщение «Ошибка при отправке. Попробуйте ещё раз.»

### Requirement: Уже подтверждено
Если гость уже подтвердил присутствие, форма MUST показывать карточку подтверждения вместо редактируемой формы.

#### Scenario: Загрузка с подтверждённым статусом
- ДАНО гость ранее подтвердил присутствие
- КОГДА RSVP-форма загружается
- ТОГДА отображается карточка с текстом «Вы подтвердили приглашение!» и «Ждём вас 12 сентября 2026»
- И поля формы не показываются
