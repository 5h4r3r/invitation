# Бэкенд
Язык: русский

## Purpose
Google Apps Script API для авторизации гостей, подтверждения присутствия и загрузки напитков.

## Requirements

### Requirement: Единая точка входа
Все API-запросы MUST проходить через единственную функцию `doGet(e)`, опубликованную как веб-приложение.

#### Scenario: Вызов API
- ДАНО фронтенд отправляет запрос
- КОГДА запрос направлен на URL `https://script.google.com/macros/s/.../exec`
- ТОГДА функция `doGet(e)` обрабатывает параметры

URL API: `https://script.google.com/macros/s/AKfycby7yLBdv3d-vNaVUOowpTydQgSphBk6wbUBZymZICIg_Sj4h5L6nxDZFm5kO_1eZ2ShgA/exec`

### Requirement: Поиск гостя по токену
Система MUST искать гостя по токену в листе «Лист1».

#### Scenario: Токен найден
- ДАНО GET-запрос с `?token=abc123`
- КОГДА токен существует в колонке A
- ТОГДА API возвращает `{ status: 'ok', name: '<имя>', confirmed: true/false }`

#### Scenario: Токен не найден
- ДАНО GET-запрос с `?token=unknown`
- КОГДА токен отсутствует во всех строках
- ТОГДА API возвращает `{ status: 'error', message: 'not found' }`

### Requirement: Подтверждение присутствия через POST
Система MUST принимать подтверждение присутствия только через HTTP POST. GET-запросы с `action=confirm` ОТКЛОНЯЮТСЯ.

#### Scenario: Подтверждение через POST
- ДАНО POST-запрос с параметрами `token=abc123&action=confirm&guests=2&drink=Вино&transfer=yes&wishes=...&allergy=...`
- КОГДА токен найден
- ТОГДА обновляются колонки C (Присутствует='yes'), D (Гостей), E (Трансфер), F (Напиток), G (Аллергия), H (Пожелания)
- И API возвращает `{ status: 'ok', message: 'confirmed' }`

#### Scenario: GET-запрос отклонён
- ДАНО GET-запрос с `?token=abc123&action=confirm`
- КОГДА запрос получен в `doGet()`
- ТОГДА возвращается `{ status: 'error', message: 'Используйте POST для подтверждения' }`
- И данные не записываются

### Requirement: Получение списка напитков
Система MUST возвращать список доступных напитков.

#### Scenario: Загрузка напитков
- ДАНО GET-запрос с `?action=getDrinks`
- КОГДА запрос обработан
- ТОГДА API возвращает `{ status: 'ok', drinks: ['Вино', 'Шампанское', ...] }`

### Requirement: Структура таблицы
Таблица с ID `1O1uKTDP6wqcsKjgClGBpzgLTBKU3zsH4xXevhF3hu0M` MUST иметь лист «Лист1» со столбцами:
A: Токен, B: Имя, C: Присутствует, D: Гостей, E: Трансфер, F: Напиток, G: Аллергия, H: Пожелания

#### Scenario: Чтение строки
- ДАНО скрипт находит строку с совпадающим токеном
- КОГДА данные читаются из листа
- ТОГДА колонки A-H отображают данные гостя

### Requirement: Серверная валидация
Система MUST проверять поля подтверждения перед записью в таблицу.

#### Scenario: Валидация guests
- ДАНО POST-запрос с `guests=10`
- КОГДА значение не в диапазоне 0-4
- ТОГДА возвращается `{ status: 'error', message: 'Некорректное количество гостей' }`
- И данные не записываются

#### Scenario: Валидация drink
- ДАНО POST-запрос с `drink=`
- КОГДА напиток пустой
- ТОГДА возвращается ошибка

#### Scenario: Валидация transfer
- ДАНО POST-запрос с `transfer=maybe`
- КОГДА значение не `yes` и не `no`
- ТОГДА возвращается ошибка

### Requirement: Сообщения об ошибках
Все сообщения об ошибках MUST быть на русском языке.

#### Scenario: Ошибка запроса
- ДАНО API возвращает ошибку
- КОГДА сообщение формируется
- ТОГДА текст ошибки на русском языке (пример: «Гость не найден», «Некорректное количество гостей»)
