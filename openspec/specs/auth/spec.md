# Аутентификация
Язык: русский

## Purpose
Идентификация гостя по токену из URL-параметра с кэшированием в localStorage.

## Requirements

### Requirement: Вход по токену
Система MUST идентифицировать гостя по уникальному токену, переданному в URL-параметре `?code=<токен>`.

#### Scenario: Первый визит с токеном
- ДАНО пользователь открывает сайт с `?code=abc123` в URL
- КОГДА страница загружается
- ТОГДА система вызывает API с `GET { token: 'abc123' }`
- И сохраняет `weddingToken`, `weddingName`, `weddingConfirmed` в localStorage
- И `?code=abc123` удаляется из URL через `window.history.replaceState`
- И имя гостя отображается в HeroSection
- И RSVP-форма показывает предыдущий статус подтверждения гостя

#### Scenario: Повторный визит без токена
- ДАНО пользователь открывает сайт без параметра `?code=`
- КОГДА страница загружается
- ТОГДА система читает `weddingToken`, `weddingName`, `weddingConfirmed` из localStorage
- И API-вызов НЕ производится
- И имя гостя отображается в HeroSection

#### Scenario: Повторный визит с новым токеном
- ДАНО пользователь с существующими данными в localStorage открывает `?code=new456`
- КОГДА страница загружается
- ТОГДА новый токен перезаписывает `weddingToken` в localStorage
- И система вызывает API с новым токеном
- И обновляет `weddingName` и `weddingConfirmed` в localStorage

#### Scenario: Нет токена нигде
- ДАНО пользователь открывает сайт без `?code=` и без данных в localStorage
- КОГДА страница загружается
- ТОГДА имя гостя отображается как запасной вариант «Гость»

### Requirement: Состояние загрузки
Система MUST показывать скелетон-заполнитель с анимацией, пока имя гостя загружается.

#### Scenario: Скелетон во время загрузки
- ДАНО пользователь с валидным токеном `?code=`
- КОГДА вызывается `initGuestName()` и API-запрос в процессе выполнения
- ТОГДА HeroSection отображает анимированную полоску скелетона вместо имени гостя
- КОГДА API отвечает
- ТОГДА скелетон заменяется именем гостя

### Requirement: Обработка ошибок
Система MUST выводить ошибки API в консоль браузера и отображать их в интерфейсе при загрузке данных гостя.

#### Scenario: Ошибка API при загрузке
- ДАНО пользователь с валидным токеном `?code=`
- КОГДА API возвращает ошибку или недоступен
- ТОГДА `apiError` устанавливается в русское сообщение об ошибке
- И ошибка логируется в консоль с префиксом «❌»
